**识别方法**：
1. 分析 Requirements.md 中的 Requirement 10（组合场景测试）
2. 审查 Phase 3 缺失场景分析报告
3. 检查已有测试覆盖（`validation-effectiveness-gate.test.ts`）
4. 基于 Deck I 卡牌能力识别潜在边界条件

**识别结果**：
- 总场景数：**8 个**
- P1 场景：**6 个**（核心边界条件，必须测试）
- P2 场景：**2 个**（次要边界条件，建议测试）

## 场景优先级分析

### P1 场景（必须测试）

| 场景 | 涉及卡牌 | 测试复杂度 | 风险等级 |
|------|---------|-----------|---------|
| 1. 弃牌能力 + 牌库为空 | 破坏者 | 低 | 高 |
| 2. 复制能力 + 无可复制目标 | 女家庭教师 | 中 | 高 |
| 3. 修正能力 + 场上无卡牌 | 图书管理员、虚空法师 | 低 | 高 |
| 4. 手牌为空 + 弃牌能力 | 伏击者、雇佣剑士 | 低 | 高 |
| 5. 印戒数量边界 | 调停者、财务官 | 中 | 高 |
| 6. 遭遇结算边界 | 审判官、调停者 | 高 | 高 |

**P1 场景特征**：
- 涉及核心游戏机制（遭遇结算、印戒、能力激活）
- 用户可能频繁遇到（牌库耗尽、手牌为空）
- 错误处理不当会导致游戏崩溃或规则错误

### P2 场景（建议测试）

| 场景 | 涉及卡牌 | 测试复杂度 | 风险等级 |
|------|---------|-----------|---------|
| 7. 持续能力 + 卡牌被移除 | 财务官、雇佣剑士、傀儡师 | 中 | 中 |
| 8. 资源耗尽 + 抽牌能力 | 精灵、学者 | 低 | 中 |

**P2 场景特征**：
- 涉及次要机制（持续能力清理、抽牌）
- 用户遇到频率较低
- 错误处理不当主要影响用户体验，不会导致游戏崩溃
