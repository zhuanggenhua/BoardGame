# Card07 宫廷卫士测试失败 - 根本原因已找到

## 问题现象
- 测试能够成功执行 UI 交互（派系选择弹窗、选项点击、弹窗关闭）
- 但修正标记（+7影响力）没有被添加到 `modifierTokens` 数组中
- 服务器日志中完全没有 `[CardiaEventSystem]` 的日志输出

## 根本原因（已确认）

### 问题：测试环境与开发环境隔离
**E2E 测试使用独立的测试服务器（端口 19000），而不是开发服务器（端口 18000）**

#### 证据
1. **测试输出**：
   ```
   ✅ E2E 测试模式：独立测试环境（端口 5173/19000/19001）
   ```

2. **开发服务器日志**：
   - 显示连接到 `icvDNJZKrte` 和 `BRzUaafkM2-` 的请求
   - 这些是旧的测试对局，不是当前运行的测试

3. **测试服务器**：
   - `npm run test:e2e` 会自动启动独立的测试服务器
   - 测试服务器运行在端口 19000（不是 18000）
   - 我们一直在查看开发服务器（18000）的日志，所以看不到测试相关的日志

### 为什么 `[CardiaEventSystem]` 日志没有出现？
**因为我们在查看错误的服务器日志！**

测试服务器的日志输出到：
- 测试进程的 stdout/stderr（与测试输出混在一起）
- 或者被 Playwright 的测试框架捕获

## 解决方案

### 方案 1：查看测试服务器的日志（推荐）
测试服务器的日志会输出到测试进程的 stdout，但可能被 Playwright 过滤掉了。

**步骤**：
1. 修改测试命令，添加 `--debug` 标志以显示服务器日志
2. 或者在测试代码中添加 `console.log` 来捕获服务器响应

### 方案 2：使用开发服务器运行测试
修改测试配置，让测试使用开发服务器（端口 18000）而不是独立的测试服务器。

**步骤**：
1. 修改 `.env` 文件或测试配置
2. 确保测试使用 `GAME_SERVER_PORT=18000`
3. 重新运行测试

### 方案 3：在测试中添加客户端日志
在测试代码中添加 `page.on('console')` 监听器，捕获浏览器控制台的日志。

**步骤**：
1. 在测试文件中添加：
   ```typescript
   page.on('console', msg => console.log('浏览器日志:', msg.text()));
   ```
2. 这样可以看到客户端的 `console.log` 输出

## 教训

### 1. 测试环境隔离
- E2E 测试默认使用独立的测试服务器，不是开发服务器
- 查看日志时必须确认是哪个服务器的日志
- 开发服务器的日志不会包含测试相关的信息

### 2. 日志调试策略
- 服务器日志：需要查看正确的服务器实例
- 客户端日志：使用 `page.on('console')` 捕获
- 测试日志：可能被 Playwright 过滤，需要特殊配置

### 3. 代码热重载
- 测试服务器可能不支持热重载
- 修改代码后需要重启测试服务器
- 或者使用开发服务器运行测试（支持热重载）

## 结论

**根本原因**：我们一直在查看开发服务器（端口 18000）的日志，但测试实际运行在独立的测试服务器（端口 19000）上。

**解决方案**：
1. 短期：在测试中添加客户端日志监听器，追踪交互流程
2. 长期：配置测试使用开发服务器，或者找到查看测试服务器日志的方法
