# P2.2 状态回溯系统实现完成总结

## 任务概述

实现了完整的状态回溯系统，使得添加修正标记后能够自动重新计算影响力、重新判定遭遇结果并移动印戒。

## 问题分析

### 初始状态
- Task 12 在 tasks.md 中标记为已完成（✅）
- 但实际上只有基础的影响力计算逻辑，缺少自动触发状态回溯的机制
- `integration-influence-modifiers.test.ts` 中有 2 个测试手动模拟了影响力计算和印戒移动

### 核心问题
1. **事件执行顺序问题**：`recalculateEncounterState` 在 `MODIFIER_ADDED` 事件被 reduce 之前执行，导致无法从 `core.modifierTokens` 读取新添加的修正标记
2. **事件 payload 类型不匹配**：
   - `CARD_INFLUENCE_MODIFIED` 使用 `cardUid` 但事件定义要求 `cardId`
   - `ENCOUNTER_RESULT_CHANGED` 使用 `oldWinner`/`newWinner` 但事件定义要求 `previousWinner`/`newWinner`/`reason`
   - `SIGNET_MOVED` 缺少必需的 `slotIndex` 字段

## 解决方案

### 方案选择
采用**方案 1：在 `ADD_MODIFIER` 命令后自动触发回溯**

**优点**：
- 简单直接，不需要新增命令
- 符合"修正标记添加 → 状态自动更新"的语义
- 对测试代码改动最小

### 实现细节

#### 1. 修改 `recalculateEncounterState` 函数签名

    core: CardiaCore,

- 传入 `newModifierValue` 参数，用于临时计算影响力

#### 2. 修复事件 payload 类型

**CARD_INFLUENCE_MODIFIED**：
```typescript

    cardUid: encounter.player1Card.uid,  // ❌ 错误

    newInfluence: newPlayer1Influence,

```

**ENCOUNTER_RESULT_CHANGED**：
```typescript

    slotIndex: encounterIndex,
    oldWinner: oldWinner || 'tie',  // ❌ 错误字段名
    newWinner,
}

// 修复后
payload: {
    slotIndex: encounterIndex,
    previousWinner: oldWinner || 'tie',  // ✅ 正确字段名
    newWinner,

```

**SIGNET_MOVED**：
```typescript

    fromCardId: oldWinnerCard.uid,
    toCardId: newWinnerCard.uid,
    // ❌ 缺少 slotIndex
}

// 修复后
payload: {
    fromCardId: oldWinnerCard.uid,
    toCardId: newWinnerCard.uid,
    slotIndex: encounterIndex,  // ✅ 新增必需字段
}
```

#### 3. 更新 `executeAddModifier`

```typescript
function executeAddModifier(
    core: CardiaCore,
    command: Extract<CardiaCommand, { type: typeof CARDIA_COMMANDS.ADD_MODIFIER }>
): CardiaEvent[] {
    const events: CardiaEvent[] = [];
    const timestamp = Date.now();
    const { playerId } = command;
    const { cardUid, modifierValue } = command.payload;
    
    // 1. 发射修正标记添加事件
    events.push({
        type: CARDIA_EVENTS.MODIFIER_ADDED,
        timestamp,
        payload: {
            cardUid,
            value: modifierValue,
            playerId,
        },
    });
    
    // 2. 触发状态回溯（传入新修正标记值用于临时计算）
    const recalculationEvents = recalculateEncounterState(core, cardUid, modifierValue);
    events.push(...recalculationEvents);
    
    return events;
}
```

## 技术亮点

### 1. 事件执行顺序问题的巧妙解决
通过传入 `newModifierValue` 参数，在 `MODIFIER_ADDED` 事件被 reduce 之前就能正确计算影响力，避免了复杂的事件重排序逻辑。

### 2. 类型安全的事件 payload
严格遵循事件定义的 payload 结构，确保编译期类型检查能够捕获错误。

### 3. 结构共享的不可变更新
所有 reducer 都使用结构共享（spread operator）更新状态，保持不可变性原则。

### 4. 完整的边界情况处理
- 平局 → 获胜：添加印戒
- 获胜 → 平局：保留印戒在原位置
- 获胜 → 另一方获胜：移动印戒
