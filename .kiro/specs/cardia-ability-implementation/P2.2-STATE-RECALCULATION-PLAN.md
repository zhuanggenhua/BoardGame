## 技术方案

### 方案 1：在 `ADD_MODIFIER` 命令后自动触发回溯（推荐）

**优点**：
- 简单直接，不需要新增命令
- 符合"修正标记添加 → 状态自动更新"的语义
- 对测试代码改动最小

**缺点**：
- `ADD_MODIFIER` 命令的职责变重（添加标记 + 触发回溯）

**实现步骤**：
1. 在 `executeAddModifier` 中，添加修正标记后，立即触发状态回溯
2. 状态回溯逻辑：
   - 找到受影响的遭遇（包含被修正卡牌的遭遇）
   - 重新计算双方卡牌的最终影响力
   - 重新判定遭遇结果
   - 如果结果改变，移动印戒
3. 产生相应的事件：
   - `CARD_INFLUENCE_MODIFIED`（影响力变化）
   - `ENCOUNTER_RESULT_CHANGED`（遭遇结果改变）
   - `SIGNET_MOVED`（印戒移动）

### 方案 2：新增 `RECALCULATE_ENCOUNTER` 命令

**优点**：
- 职责分离，`ADD_MODIFIER` 只负责添加标记
- 更灵活，可以手动触发回溯

**缺点**：
- 需要新增命令类型
- 测试代码需要手动调用 `RECALCULATE_ENCOUNTER`
- 增加了系统复杂度

**实现步骤**：
1. 在 `commands.ts` 中新增 `RECALCULATE_ENCOUNTER` 命令
2. 在 `execute.ts` 中实现 `executeRecalculateEncounter`
3. 测试代码需要在 `ADD_MODIFIER` 后手动调用 `RECALCULATE_ENCOUNTER`

#### 1. 修改 `execute.ts` 中的 `executeAddModifier`

```typescript
function executeAddModifier(
    core: CardiaCore,
    command: Extract<CardiaCommand, { type: typeof CARDIA_COMMANDS.ADD_MODIFIER }>
): CardiaEvent[] {
    const events: CardiaEvent[] = [];
    const timestamp = Date.now();
    const { playerId } = command;
    const { cardUid, modifierValue } = command.payload;
    
    // 1. 发射修正标记添加事件
    events.push({
        type: CARDIA_EVENTS.MODIFIER_ADDED,
        timestamp,
        payload: {
            cardUid,
            value: modifierValue,
            playerId,
        },
    });
    
    // 2. 触发状态回溯
    const recalculationEvents = recalculateEncounterState(core, cardUid);
    events.push(...recalculationEvents);
    
    return events;
}
```

#### 2. 新增 `recalculateEncounterState` 函数

```typescript
/**
 * 重新计算遭遇状态
 * 当修正标记添加后，重新计算影响力、遭遇结果和印戒位置
 */
function recalculateEncounterState(
    core: CardiaCore,
    affectedCardUid: string
): CardiaEvent[] {
    const events: CardiaEvent[] = [];
    const timestamp = Date.now();
    
    // 1. 找到受影响的遭遇（包含被修正卡牌的遭遇）
    const encounterIndex = core.encounterHistory.findIndex(encounter => 
        encounter.player1Card.uid === affectedCardUid || 
        encounter.player2Card.uid === affectedCardUid
    );
    
    if (encounterIndex === -1) {
        // 卡牌不在任何遭遇中，无需回溯
        return events;
    }
    
    const encounter = core.encounterHistory[encounterIndex];
    
    // 2. 重新计算双方卡牌的最终影响力
    const player1Modifiers = core.modifierTokens.filter(t => t.cardId === encounter.player1Card.uid);
    const player2Modifiers = core.modifierTokens.filter(t => t.cardId === encounter.player2Card.uid);
    
    const newPlayer1Influence = player1Modifiers.reduce(
        (acc, m) => acc + m.value, 
        encounter.player1Card.baseInfluence
    );
    const newPlayer2Influence = player2Modifiers.reduce(
        (acc, m) => acc + m.value, 
        encounter.player2Card.baseInfluence
    );
    
    // 3. 发射影响力变化事件
    if (newPlayer1Influence !== encounter.player1Influence) {
        events.push({
            type: CARDIA_EVENTS.CARD_INFLUENCE_MODIFIED,
            timestamp,
            payload: {
                cardUid: encounter.player1Card.uid,
                oldInfluence: encounter.player1Influence,
                newInfluence: newPlayer1Influence,
            },
        });
    }
    
    if (newPlayer2Influence !== encounter.player2Influence) {
        events.push({
            type: CARDIA_EVENTS.CARD_INFLUENCE_MODIFIED,
            timestamp,
            payload: {
                cardUid: encounter.player2Card.uid,
                oldInfluence: encounter.player2Influence,
                newInfluence: newPlayer2Influence,
            },
        });
    }
    
    // 4. 重新判定遭遇结果（考虑持续能力）
    let newWinner: PlayerId | 'tie';
    
    if (newPlayer1Influence > newPlayer2Influence) {
        newWinner = encounter.player1Card.ownerId;
    } else if (newPlayer2Influence > newPlayer1Influence) {
        newWinner = encounter.player2Card.ownerId;
    } else {
        newWinner = 'tie';
    }
    
    // 应用持续能力效果
    const mediatorAbility = core.ongoingAbilities.find(
        a => a.effectType === 'forceTie' && a.encounterIndex === encounterIndex
    );
    if (mediatorAbility) {
        newWinner = 'tie';
    }
    
    const magistrateAbility = core.ongoingAbilities.find(
        a => a.effectType === 'winTies'
    );
    if (magistrateAbility && newWinner === 'tie') {
        newWinner = magistrateAbility.playerId;
    }
    
    // 5. 如果遭遇结果改变，发射事件并移动印戒
    const oldWinner = encounter.winnerId;
    
    if (oldWinner !== newWinner) {
        // 发射遭遇结果改变事件
        events.push({
            type: CARDIA_EVENTS.ENCOUNTER_RESULT_CHANGED,
            timestamp,
            payload: {
                slotIndex: encounterIndex,
                oldWinner: oldWinner || 'tie',
                newWinner,
            },
        });
        
        // 移动印戒
        if (oldWinner && oldWinner !== 'tie') {
            const oldWinnerCard = oldWinner === encounter.player1Card.ownerId 
                ? encounter.player1Card 
                : encounter.player2Card;
            
            if (newWinner !== 'tie') {
                const newWinnerCard = newWinner === encounter.player1Card.ownerId 
                    ? encounter.player1Card 
                    : encounter.player2Card;
                
                // 从旧获胜卡牌移动印戒到新获胜卡牌
                events.push({
                    type: CARDIA_EVENTS.SIGNET_MOVED,
                    timestamp,
                    payload: {
                        fromCardId: oldWinnerCard.uid,
                        toCardId: newWinnerCard.uid,
                    },
                });
            } else {
                // 新结果是平局，移除印戒（不移动到任何卡牌）
                // 注意：这种情况下印戒应该如何处理？
                // 根据游戏规则，平局时印戒应该保留在原位置还是移除？
                // 暂时假设保留在原位置（不产生 SIGNET_MOVED 事件）
            }
        } else if (newWinner !== 'tie') {
            // 旧结果是平局，新结果有获胜者，添加印戒
            const newWinnerCard = newWinner === encounter.player1Card.ownerId 
                ? encounter.player1Card 
                : encounter.player2Card;
            
            events.push({
                type: CARDIA_EVENTS.EXTRA_SIGNET_PLACED,
                timestamp,
                payload: {
                    cardId: newWinnerCard.uid,
                    playerId: newWinner,
                },
            });
        }
    }
    
    return events;
}
```

## 风险评估

### 风险 1：遭遇历史不可变性
**问题**：遭遇历史是不可变的，修改历史中的影响力和获胜者可能违反不可变性原则
**解决方案**：使用结构共享，创建新的遭遇对象替换旧的

### 风险 2：印戒移动的边界情况
**问题**：平局 → 获胜、获胜 → 平局、获胜 → 另一方获胜，三种情况的印戒处理逻辑不同
**解决方案**：详细测试所有边界情况，确保印戒移动逻辑正确

### 风险 3：持续能力的影响
**问题**：调停者（强制平局）和审判官（赢得平局）会影响遭遇结果判定
**解决方案**：在重新判定时考虑持续能力效果
