## 二、规则图片关键信息提取

### 2.1 游戏目标和玩法（第3页）

**游戏目标：**
- 卡迪亚是一款对战游戏
- 每一回合，你们打出2张牌互相对抗
- 这称为一次遭遇
- 影响力更高的牌赢得一枚印戒
- 影响力较低的牌则会发动它的能力

**关键规则：**
- 打出的牌会留在场上，并且可以受到能力的影响
- 已获得印戒在游戏过程中可能会有更详细的说明
- 按照游戏玩法，将后续的牌放在之前的遭遇旁边，使它们继续相对而置，如此进行下去
- 第一个在自己的牌上集齐5枚印戒的玩家赢得游戏

**建议：**
- 我们建议游玩三局两胜（最好五局三胜）

---

### 2.2 回合概览（第3页）

一局游戏包含若干回合，直到一名玩家获胜为止。每回合由三个阶段组成：

1. **打牌**
2. **发动能力**
3. **回合结束**

---

### 2.3 打牌阶段（第3页）

**流程：**
- 双方各自从手牌中选择1张牌，面朝下放在自己场区
- 选择完成后，同时翻开比较

**放置牌：**
- 从你的手牌中选择1张牌，面朝下放在你的场区
- 你的对手也选择1张牌，面朝下放在与你的牌对应的位置

**揭示牌：**
- 现在同时揭示你们的牌，并比较它们的影响力
- 影响力更高的牌赢得这次遭遇，从补给区拿取1枚印戒，放在获胜的那张牌上

**平局：**
- 如果两张牌的影响力相同，则这次遭遇为平局，没有牌赢或输
- 因此不会放印戒，跳过阶段2（发动能力），直接进入阶段3（回合结束）

---

### 2.4 发动能力阶段（第4页）

**核心规则：**
> **重要：只有当前遭遇中输掉的那张牌的能力会被发动。如果这个能力导致一张已打出的牌在过去的遭遇中失利，这张牌的能力不会因此发动。**

**注意事项：**
- 有时输掉遭遇并非发动能力，反而对你更有利
- 你必须尽可能完整地算发动的能力，包括对其他牌造成的后果
- 结算能力时，按照牌上的文字执行
- 注意：即使你无法完全或根本无法结算某个能力，你仍然可以打出该牌

**重要：**
- 有些能力会与游戏的通用规则相冲突
- 在这种情况下，能力的效果永远优先于通用规则
- 游戏中有两种能力，每种都有不同的图标：⚡ 即时能力和 🔄 持续能力
- 有些能力可以改变牌的影响力（见第10-11页）

---

### 2.5 即时能力（第5页）

**⚡ 即时能力：**
- 当你发动一项即时能力时，立刻结算一次

**示例：**
- 你的破坏者(4)在这次遭遇中失利，因此你发动了他的能力
- 你的对手从其牌库顶弃2张牌
- 你的对手将这2张牌面朝上放入其弃牌堆

---

### 2.6 持续能力（第5页）

**🔄 持续能力：**
- 当你发动一项持续能力时，在该牌上放置一个持续标记🔄
- 从现在开始，这项持续能力生效
- 即使该牌在之后的游戏中获胜，这项持续能力仍然会保持生效
- 如果持续标记从牌上被移除，则持续能力立即失效
- 一旦持续能力失效，立刻结算因此产生的变化（包括影响到的过去遭遇）

**示例（第5页底部）：**
在第二次遭遇中，你的对手再次获胜，并在其牌上放置1枚印戒。

你发动你的调停者(4)的能力：这次遭遇为平局。

你在调停者上放置一个持续标记。只要该标记存在，这次遭遇就视为平局。因此，你需要将对手牌上的那枚印戒归还到补给区。

**注意：**
- 有些能力允许你在已经打出的牌上放置持续标记
- 这意味着你可以激活一张过去遭遇中的牌的持续能力

**特殊情况：**
- 标记🔄和🎯数量不限
- 如果用完了，可以用任何合适的替代物

```typescript
const isTie = encounterResolvedEvent && 
              !encounterResolvedEvent.payload.winnerId && 
              !encounterResolvedEvent.payload.loserId;

events.push({
    type: CARDIA_EVENTS.PHASE_CHANGED,
    timestamp: Date.now(),
    payload: {
        oldPhase: 'play',
        newPhase: isTie ? 'end' : 'ability',  // ✅ 平局跳过能力阶段
    },
});
```

**结论：** ✅ 完全正确

### 3.2 能力发动限制 ✅

**规则：**

**当前实现：**

```typescript
if (core.currentEncounter.loserId !== playerId) {
    return { valid: false, error: 'Only the loser can activate abilities' };

    return core.currentEncounter.loserId || core.currentPlayerId;
```

**结论：** ✅ 完全正确

### 3.3 历史遭遇能力不触发 ✅

**规则：**
> 如果这个能力导致一张已打出的牌在过去的遭遇中失利，这张牌的能力不会因此发动。

**当前实现：**

- 没有机制允许重新激活过去遭遇的能力
- 持续能力通过标记系统管理，不会重新触发

**结论：** ✅ 正确实现

### 3.4 能力优先于规则 ✅

**规则：**
> 有些能力会与游戏的通用规则相冲突。在这种情况下，能力的效果永远优先于通用规则。

**当前实现：**
- 能力通过事件系统执行，可以修改任何状态
- 没有硬编码的规则限制能力效果
- 例如：调停者能力可以将遭遇结果改为平局，移除已授予的印戒

**结论：** ✅ 架构支持

### 3.5 持续能力机制 ✅

**规则：**
- 持续能力需要放置持续标记
- 标记存在时效果持续生效
- 标记移除时效果立即失效

**当前实现：**
```typescript
// 使用 TagContainer 管理持续标记
player.tags.tags[`Ongoing.${abilityId}`]

// 添加持续效果

// 移除持续效果
removeTag(player.tags, `Ongoing.${abilityId}`)
```

**结论：** ✅ 正确实现
