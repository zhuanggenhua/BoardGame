# 验证图片加载优化

## 快速验证步骤

### 1. 重启开发服务器
```bash
# 停止当前服务器（Ctrl+C）
# 重新启动
npm run dev
```

**为什么需要重启？** 新增的 `assets.ts` 文件需要重新加载才能生效。

### 2. 打开浏览器控制台
按 `F12` 打开开发者工具，切换到 **Console** 面板。

### 3. 访问游戏
在浏览器中访问 `http://localhost:3000`，创建 Cardia 游戏房间。

### 4. 检查资源注册日志
在控制台中应该看到：
```
[Cardia] 资源已注册，共 43 张关键图片
```

**如果没有看到这条日志：**
- 确认已重启开发服务器
- 检查 `src/games/cardia/game.ts` 是否包含 `import './assets';`
- 检查浏览器控制台是否有错误信息

### 5. 检查预加载执行
游戏开始前（LoadingScreen 显示时），控制台应该显示：
```
[AssetLoader] cardia 关键图片预加载耗时 XXXms（43 张）
```

**预加载时间参考：**
- 本地环境（首次）：1000-3000ms
- 本地环境（缓存）：< 100ms
- CDN（首次）：3000-10000ms
- CDN（缓存）：< 500ms

**如果预加载时间过长（> 10s）：**
- 检查网络连接
- 检查是否使用了 CDN（`.env` 中的 `VITE_ASSETS_BASE_URL`）
- 尝试禁用 CDN，使用本地资源

### 6. 检查卡牌加载日志
游戏开始后，手牌区域显示卡牌时，控制台应该显示：
```
[CardDisplay] 加载图片: { cardUid: "...", defId: "deck_i_card_01", imagePath: "cardia/cards/deck1/1", loaded: false, error: false }
[CardDisplay] 图片加载成功: deck_i_card_01 cardia/cards/deck1/1
```

**每张卡牌应该有两条日志：**
1. 第一条：开始加载（loaded: false）
2. 第二条：加载成功

**如果看到加载失败的日志：**
```
[CardDisplay] 图片加载失败: deck_i_card_01 cardia/cards/deck1/1
```
说明预加载没有生效或图片文件不存在，请检查：
- 运行 `node scripts/test-cardia-image-paths.mjs` 验证文件存在
- 检查 Network 面板中的 404 请求

### 7. 检查视觉指示器
观察卡牌右上角：
- **无圆点**：图片加载成功 ✅
- **黄色圆点（闪烁）**：图片正在加载中 ⚠️
- **红色圆点**：图片加载失败 ❌

**理想状态：** 所有卡牌都应该无圆点（预加载已完成，直接从缓存读取）

**如果看到黄色圆点：**
- 说明预加载没有覆盖这张卡牌
- 检查 `assets.ts` 中的 `criticalImages` 列表是否完整

**如果看到红色圆点：**
- 说明图片加载失败
- 检查 Network 面板中的失败请求
- 检查文件是否存在

### 8. 检查网络请求
切换到 **Network** 面板：

1. 刷新页面
2. 在过滤器中输入 `.webp`
3. 查看所有图片请求

**期望结果：**
- ✅ 所有请求状态码 `200 OK`
- ✅ 大部分请求显示 `(from disk cache)` 或 `(from memory cache)`
- ✅ 加载时间 < 10ms（缓存命中）

**如果看到很多请求没有缓存标记：**
- 可能是首次加载，刷新页面再试
- 检查浏览器缓存设置（开发者工具 → Network → Disable cache 应该未勾选）

## 成功标准

### ✅ 优化成功的标志
1. 控制台显示 "资源已注册，共 43 张关键图片"
2. 控制台显示 "关键图片预加载耗时 XXXms"
3. 所有卡牌立即显示，无黄色或红色圆点
4. 所有卡牌加载日志显示 "图片加载成功"
5. Network 面板显示大部分请求来自缓存
6. 刷新页面多次，卡牌始终正常显示（稳定性）

### ❌ 仍有问题的标志
1. 控制台没有 "资源已注册" 日志 → 重启服务器
2. 控制台没有 "预加载耗时" 日志 → 检查 AssetLoader 配置
3. 卡牌显示黄色圆点 → 预加载未覆盖
4. 卡牌显示红色圆点 → 图片加载失败
5. 控制台显示 "图片加载失败" → 检查文件路径
6. Network 面板显示 404 → 运行路径验证脚本

## 常见问题排查

### Q1: 控制台没有 "资源已注册" 日志
**原因：** `assets.ts` 没有被执行

**解决：**
1. 确认 `src/games/cardia/game.ts` 包含 `import './assets';`
2. 重启开发服务器（Ctrl+C 后重新 `npm run dev`）
3. 清除浏览器缓存并刷新页面

### Q2: 预加载时间过长（> 10s）
**原因：** 网络慢或 CDN 不稳定

**解决：**
1. 检查网络连接
2. 临时禁用 CDN：
   ```bash
   # .env 文件中注释掉
   # VITE_ASSETS_BASE_URL=https://assets.easyboardgame.top/official
   ```
3. 重启开发服务器

### Q3: 部分卡牌仍然加载失败
**原因：** 文件路径错误或文件不存在

**解决：**
```bash
# 运行路径验证脚本
node scripts/test-cardia-image-paths.mjs

# 应该显示：✅ 通过: 35/35
```

如果验证失败，检查文件是否存在：
```bash
ls public/assets/i18n/zh-CN/cardia/cards/deck1/compressed/
ls public/assets/i18n/zh-CN/cardia/cards/deck2/compressed/
```

### Q4: 刷新后仍然不稳定
**原因：** 浏览器缓存问题

**解决：**
1. 清除浏览器缓存（Ctrl+Shift+Delete）
2. 使用无痕模式测试（Ctrl+Shift+N）
3. 检查开发者工具 Network 面板的 "Disable cache" 是否勾选（应该不勾选）

## 性能对比

### 优化前
- 首次加载：每张卡牌 200-500ms，总计 1-2.5s
- 并发限制：超过 6 张卡牌时出现排队
- 失败率：10-30%（随机）
- 用户体验：卡牌逐个显示，部分失败

### 优化后
- 预加载：游戏开始前完成，1-3s（一次性）
- 游戏内加载：< 10ms（缓存命中）
- 失败率：0%（稳定）
- 用户体验：所有卡牌立即显示，流畅
