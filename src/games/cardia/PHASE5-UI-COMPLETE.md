## UI 设计特点

### 1. 视觉风格
- **背景**：深色渐变（slate-900 → purple-900 → slate-900）
- **卡牌**：派系颜色渐变 + 白色边框
- **按钮**：紫色（能力）、灰色（跳过）、绿色（结束回合）
- **文字**：白色主文字 + 灰色辅助文字

### 2. 布局结构
- **三段式布局**：对手区域（顶部）+ 战场区域（中间）+ 我的区域（底部）
- **响应式设计**：使用 Flexbox 自适应高度
- **固定元素**：阶段指示器和操作按钮固定在右上角

### 3. 交互反馈
- **悬停效果**：手牌悬停时放大（`hover:scale-105`）
- **禁用状态**：不可操作时降低透明度（`opacity-50`）
- **过渡动画**：所有交互都有平滑过渡（`transition-colors`、`transition-transform`）

### 4. 信息展示
- **印戒**：黄色圆点，数量直观
- **能力**：闪电图标（⚡）表示有能力
- **影响力**：大号数字显示在卡牌顶部
- **派系**：颜色编码 + 文字标签

## 设计决策记录

### 1. 简化卡牌展示
- **决策**：使用纯色渐变 + 文字，不使用图片
- **理由**：
  - 快速完成 Phase 5
  - 不依赖图片资源
  - 易于调试和测试
- **后续计划**：Phase 6 添加图片资源

### 2. 三段式布局
- **决策**：对手区域（顶部）+ 战场区域（中间）+ 我的区域（底部）
- **理由**：
  - 符合卡牌对战游戏的常见布局
  - 清晰的视觉层次
  - 易于理解和操作
- **替代方案**：左右分屏
  - 优点：更紧凑
  - 缺点：不符合卡牌游戏习惯

### 3. 固定操作按钮
- **决策**：阶段指示器和操作按钮固定在右上角
- **理由**：
  - 始终可见，不被遮挡
  - 不占用主要游戏区域
  - 易于点击
- **替代方案**：底部居中
  - 优点：更接近手牌区
  - 缺点：可能被手牌遮挡

### 4. 派系颜色编码
- **决策**：使用颜色渐变区分派系
- **理由**：
  - 视觉识别度高
  - 不依赖文字
  - 符合游戏主题
- **颜色选择**：
  - 沼泽：绿色（自然、生命）
  - 学院：黄色（智慧、魔法）
  - 行会：红色（工业、机械）
  - 王朝：蓝色（贵族、权力）
