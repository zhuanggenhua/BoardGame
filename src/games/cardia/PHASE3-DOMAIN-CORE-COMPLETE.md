## 核心游戏流程

### 回合流程
1. **阶段1：打出卡牌（play）**
   - 双方玩家同时选择1张手牌
   - 执行 `PLAY_CARD` 命令
   - 当双方都打出卡牌后，自动解析遭遇战
   - 进入阶段2

2. **阶段2：激活能力（ability）**
   - 失败者可以激活其卡牌上的能力
   - 执行 `ACTIVATE_ABILITY` 命令或 `SKIP_ABILITY` 命令
   - 进入阶段3

3. **阶段3：回合结束（end）**
   - 双方玩家各抽1张牌
   - 执行 `END_TURN` 命令
   - 回到阶段1，开始新回合

### 遭遇战解析
1. 计算双方卡牌的最终影响力（基础值 + 修正标记）
2. 比较影响力，判定胜负
3. 胜者在其卡牌上获得1个印戒
4. 平局：双方都不获得印戒，跳过能力阶段

### 游戏结束条件
1. 某玩家获得5个印戒（默认）
2. 某玩家无法打出卡牌（手牌和牌库均为空）
3. 某能力直接宣告胜利（如精灵、机械精灵）

## 设计决策记录

### 1. 命令与事件分离
- **命令**：玩家意图（如"打出卡牌"）
- **事件**：已发生的事实（如"卡牌已打出"）
- **好处**：清晰的因果关系，易于测试和回放

### 2. 结构共享
- 使用 spread 操作符只修改变更的部分
- 保持不变部分的引用
- **好处**：性能优化，支持时间旅行调试

### 3. 阶段驱动
- 游戏流程由阶段（play → ability → end）驱动
- 每个阶段只允许特定的命令
- **好处**：简化验证逻辑，防止非法操作

### 4. 遭遇战自动解析
- 当双方都打出卡牌后，自动解析遭遇战
- 不需要额外的"解析遭遇战"命令
- **好处**：简化玩家操作，符合游戏规则

### 5. 失败者激活能力
- 只有失败者可以激活能力
- 平局时跳过能力阶段
- **好处**：符合游戏规则，简化逻辑
