# Change: 音频设置入库与跨设备同步

## Why
当前音频设置（静音/音量）仅保存在本地，登录用户在跨设备/跨浏览器时无法同步，清理缓存会丢失配置。

## What Changes
- 新增用户音频设置持久化存储与读取接口（登录态）。
- 前端在登录后拉取服务器设置并应用；未登录继续使用本地缓存。
- 首次登录且服务器无设置时，将本地设置迁移到服务器。
- 更新音量时同时写入服务器与本地缓存。
- 补充后端/前端测试覆盖同步与迁移流程。

## Impact
- Affected specs: manage-user-settings（新增能力）
- Affected code: apps/api 用户设置模块、前端音频管理与登录同步流程、测试用例
